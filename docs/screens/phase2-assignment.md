### 과제 vs 라이브 차이점

| 항목 | 라이브 (S-06) | 과제 (S-11) |
|------|--------------|-------------|
| 문항 전환 | 서버 트리거 (Host 컨트롤) | 학생이 직접 [다음 문항] 버튼 클릭 |
| WebSocket | 필수 (실시간) | 불필요 (REST만 사용) |
| 일시정지 | Host가 강제 | 없음 |
| 종료 | 자동 (Host 트리거) | 학생이 직접 [제출하기] |
| 이어하기 | 없음 | 있음 (진입 시 팝업) |
| 제출 확인 | 없음 | 있음 (제출 전 팝업) |

### 상태 분기

| 상태 | UI 처리 |
|------|---------|
| Member 이어하기 | 진입 시 이전 진행률 팝업 (이어하기 / 처음부터) |
| Guest 이어하기 | 이메일+닉네임 입력 후 동일 인물 확인 → 이어하기 팝업 |
| 마감된 과제 접속 | "마감된 과제입니다" 안내 후 `/assignments` 이동 |
| 이미 제출한 과제 접속 | 결과 화면으로 리다이렉트 |
| 미응답 문항 포함 제출 | 팝업에 미응답 문항 수 표시 |

### WebSocket

과제 모드는 WebSocket 불필요. REST API만 사용.

### API

```
GET  /api/sessions/[sessionId]/join         → Session 기본 정보 + 진행 상태
POST /api/sessions/[sessionId]/guest-enter  → GuestSession (게스트 입장)
POST /api/sessions/[sessionId]/answer       → { questionId, answer, timeSpent }
POST /api/sessions/[sessionId]/submit       → 204 (최종 제출)
```
