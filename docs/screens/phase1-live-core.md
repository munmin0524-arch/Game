# Phase 1 — 라이브 핵심 화면 스펙

> 기술 스택: Next.js 14 + TypeScript + Tailwind CSS + shadcn/ui
> 대상 역할: Host(교사) + 학생(Member/Guest)
> 실시간 통신: WebSocket (Socket.io)

---

## 화면 목록

| # | 화면명 | 라우트 | 역할 |
|---|--------|--------|------|
| S-01 | 홈 대시보드 | `/dashboard` | Host |
| S-02 | 세트지 목록 | `/sets` | Host |
| S-03 | 에디터 | `/sets/[setId]/edit` | Host |
| S-04 | 배포 설정 (라이브) | `/sets/[setId]/deploy` | Host |
| S-05 | QR 대기화면 | `/live/[sessionId]/waiting` | Host |
| S-06 | 플레이 화면 | `/play/[sessionId]` | 학생 |
| S-07 | 컨트롤 패널 | `/live/[sessionId]/control` | Host |
| S-08 | 결과·랭킹 화면 | `/result/[sessionId]` | Host + 학생 |

### 레이아웃

```
┌─────────────────────────────────────────────────────┐
│  세트지명                            [강제 종료 ✕]   │
├──────────────────────┬──────────────────────────────┤
│                      │  입장한 학생 (실시간)          │
│   ┌──────────────┐   │  ┌──────────────────────────┐ │
│   │              │   │  │ 👤 홍길동    [Member]    │ │
│   │   QR CODE    │   │  │ 👤 김철수    [Guest]     │ │
│   │   (256×256)  │   │  │ 👤 이영희    [Member]    │ │
│   │              │   │  │    ···                    │ │
│   └──────────────┘   │  └──────────────────────────┘ │
│   [링크 복사 📋]      │  총 {N}명 입장              │ │
│   [QR 재생성 🔄]      │                              │
│                      │  [학생 강퇴] (선택 후 활성)   │
├──────────────────────┴──────────────────────────────┤
│                                                     │
│          [게임 시작 →]   (학생 0명 시 비활성)        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 상태 분기

| 상태 | UI 처리 |
|------|---------|
| 학생 0명에서 시작 클릭 | Dialog: "아직 입장한 학생이 없어요. 그래도 시작할까요?" |
| QR 재생성 클릭 | Confirm Dialog → 재생성 후 QR 이미지 교체 |
| 학생 강퇴 | Confirm Dialog → WebSocket `kick` 이벤트 발송 |
| WebSocket 끊김 | Toast: "연결이 끊겼습니다. 재연결 중..." + 자동 재연결 |

### WebSocket 이벤트

```typescript
// 수신 (서버 → 클라이언트)
'student:joined'   → { participant: Participant }   // 학생 입장
'student:left'     → { participantId: string }      // 학생 이탈

// 발신 (클라이언트 → 서버)
'session:start'    → { sessionId: string }          // 게임 시작
'student:kick'     → { participantId: string }      // 강퇴
'qr:regenerate'    → { sessionId: string }          // QR 재생성
```

### API

```
GET  /api/sessions/[sessionId]         → Session + 현재 참여자 목록
POST /api/sessions/[sessionId]/start   → 204
POST /api/sessions/[sessionId]/qr      → { qrCode: string, expiresAt: string }
```

// 수신 (서버 → 클라이언트)

'game:pause'       → {}                                          // 일시정지
'game:resume'      → {}                                          // 재개
'game:end'         → { sessionId: string }                       // 종료

// 발신 (클라이언트 → 서버)
'answer:submit'    → { questionId: string, answer: string, timeSpent: number }

| 마지막 문항에서 건너뛰기 | Confirm: "마지막 문항입니다. 건너뛰면 게임이 종료됩니다." |
| 힌트 미설정 문항에서 힌트 공개 | Toast: "이 문항에는 힌트가 없습니다" |
| 강제 종료 | Confirm Dialog (취소 가능) |

### WebSocket 이벤트

```typescript
// 발신 (Host → 서버 → 전체 학생)
'game:pause'         → {}
'game:resume'        → {}
'question:skip'      → { sessionId: string }
'time:extend'        → { seconds: 30 }
'hint:reveal'        → { questionId: string }
'game:force-end'     → { sessionId: string }

// 수신 (서버 → Host)
'answer:count-update' → { answered: number, total: number, distribution: Record<string, number> }
```

### API

```
GET  /api/sessions/[sessionId]/control    → Session + 현재 진행 상태
POST /api/sessions/[sessionId]/force-end  → 204
```
