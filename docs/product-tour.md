## 1. 개요

### 목적
교육 퀴즈 게임 플랫폼(QUIZ PARTY)의 유저 플로우와 화면별 기능을 **기획서/보고용**으로 시연하기 위한 가이드 시스템.

### 핵심 개념
- **딤 처리(Dim Overlay)**: 화면 전체를 어둡게 처리
- **스포트라이트(Spotlight)**: 설명할 UI 요소만 밝게 하이라이트 (둥근 모서리 컷아웃)
- **툴팁 말풍선(Tooltip)**: 하이라이트된 요소 옆에 설명 + 스텝 네비게이션

### 두 가지 모드

| 모드 | 설명 | 동작 방식 |
|------|------|----------|
| **#1 화면별 설명** | 특정 화면의 기능/구성요소를 스텝별로 설명 | 해당 화면으로 이동 → 딤 + 스포트라이트 투어 |
| **#2 유저 플로우** | 여러 화면을 순서대로 이동하며 전체 사용 흐름 설명 | 화면A 투어 → "다음" → 화면B로 이동 → 투어 반복 |

### 대상 사용자
- CP, 상사 등 내부 이해관계자 (기획 리뷰/보고 시)
- 추후 실제 서비스에 온보딩 가이드로 탑재 가능

## 2. 투어 허브 (진입점)

### 진입 방법
- **GNB "📖 가이드" 버튼** 클릭 → 투어 허브 오버레이 표시
- **URL 파라미터** `?tour=hub` → 페이지 로드 시 자동으로 허브 표시

### 투어 허브 레이아웃 (2컬럼)

```
┌──────────────────────────────────────────────────────────────┐
│                                                         [✕]  │
│                                                              │
│  📖 서비스 가이드                                             │
│  QUIZ PARTY의 주요 화면과 사용 흐름을 안내합니다               │
│                                                              │
│  ┌──────────────────────────┐  ┌───────────────────────────┐ │
│  │  #1 화면별 설명           │  │  #2 유저 플로우            │ │
│  │                          │  │                           │ │
│  │  교사용 ──────────────   │  │  ① 퀴즈 만들기 → 게임     │ │
│  │   1) 대시보드             │  │    새 퀴즈 생성부터        │ │
│  │   2) 내 퀴즈              │  │    라이브 게임까지         │ │
│  │   3) 에디터               │  │                           │ │
│  │   4) 게임 설정            │  │  ② 퀴즈 광장 → 게임       │ │
│  │   5) QR 대기화면          │  │    다른 선생님 퀴즈를      │ │
│  │   6) 컨트롤 패널          │  │    가져와서 바로 활용      │ │
│  │   7) 그룹 관리            │  │                           │ │
│  │   8) 퀴즈 광장            │  │  ③ 그룹 관리 → 게임       │ │
│  │   9) 지난 게임            │  │    반/그룹 만들고 게임     │ │
│  │  10) 리포트 (문항/학생)   │  │                           │ │
│  │                          │  │  ④ 과제 보내기             │ │
│  │  학생용 ──────────────   │  │    비동기 과제 발송/관리   │ │
│  │  11) 플레이               │  │                           │ │
│  │  12) 결과 / 랭킹          │  │  ⑤ 리포트 분석            │ │
│  │                          │  │    게임 결과 심층 분석     │ │
│  │                          │  │                           │ │
│  │                          │  │  ⑥ AI로 퀴즈 만들기       │ │
│  │                          │  │    AI 추천으로 빠르게 생성 │ │
│  └──────────────────────────┘  └───────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 동작 규칙
- **화면 항목 클릭** → 허브 닫기 → 해당 페이지로 이동(`router.push`) → 도착 후 자동으로 딤 + 스포트라이트 투어 시작
- **유저 플로우 항목 클릭** → 허브 닫기 → 플로우 첫 번째 페이지로 이동 → 멀티 페이지 투어 시작
- **✕ 버튼 또는 ESC** → 허브 닫기, 아무 일도 없음
- **딤 영역 클릭** → 허브 닫기

### 6-1. 타입 정의

```typescript
// 단일 스텝
interface TourStep {
  target: string              // data-tour 속성값 (예: "dashboard-create")
  title: string               // 스텝 제목
  description: string         // 설명 텍스트
  placement?: 'top' | 'bottom' | 'left' | 'right'  // 툴팁 위치 (기본: bottom)
  spotlightPadding?: number   // 스포트라이트 패딩 (기본: 8px)
  spotlightRadius?: number    // 스포트라이트 둥근 모서리 (기본: 12px)
}

// 유저 플로우의 한 페이지
interface TourFlowPage {
  route: string               // 이동할 라우트 (예: "/sets/demo/edit")
  pageTitle: string           // 화면 이름 (예: "에디터")
  steps: TourStep[]           // 이 페이지에서의 스텝들
}

// 유저 플로우 전체
interface TourFlow {
  id: string                  // 고유 ID (예: "quiz-to-game")
  title: string               // 플로우 제목 (예: "퀴즈 만들기 → 라이브 게임 시작")
  description: string         // 한줄 설명
  pages: TourFlowPage[]       // 페이지 목록 (순서대로)
}

// 투어 상태
type TourMode =
  | { type: 'idle' }
  | { type: 'hub' }
  | { type: 'screen'; steps: TourStep[]; currentStep: number }
  | { type: 'flow'; flow: TourFlow; pageIndex: number; stepIndex: number }

// Context 값
interface TourContextValue {
  mode: TourMode
  openHub: () => void
  startScreenTour: (steps: TourStep[]) => void
  startFlowTour: (flow: TourFlow) => void
  nextStep: () => void
  prevStep: () => void
  endTour: () => void
}
```

### 6-6. 요소 미발견 시 처리

MutationObserver로 최대 2초 대기 후 스킵:

```typescript
function waitForElement(target: string, timeout = 2000): Promise<Element | null> {
  return new Promise((resolve) => {
    const el = document.querySelector(`[data-tour="${target}"]`)
    if (el) return resolve(el)

    const observer = new MutationObserver(() => {
      const found = document.querySelector(`[data-tour="${target}"]`)
      if (found) { observer.disconnect(); resolve(found) }
    })
    observer.observe(document.body, { childList: true, subtree: true })

    setTimeout(() => { observer.disconnect(); resolve(null) }, timeout)
  })
}
```
