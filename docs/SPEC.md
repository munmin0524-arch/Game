## 1. 사용자 역할

| 역할 | 설명 | 식별 방식 | 접근 가능 화면 |
|------|------|-----------|---------------|
| **Host (교사)** | 퀴즈 제작, 게임 운영, 리포트 조회 | 로그인 (UUID) | 대시보드 전체 |
| **Member (학생 계정)** | QR/링크로 참여. 학습 데이터 누적 | 로그인 (UUID) | 과제 목록 + 플레이 + 결과 |
| **Guest (비회원 학생)** | 가입 없이 QR/링크로 참여 | 이메일 + 닉네임 | 플레이 + 결과만 |

**핵심 규칙**
- Host는 대시보드에만 직접 접근. Member/Guest는 QR/링크로만 진입.
- 게스트가 동일 이메일+닉네임으로 재접속 시 동일 인물로 이어받기 (기기/브라우저 무관).
- 게스트 이메일로 회원가입 시 기존 게스트 데이터가 Member에게 자동 연결.

## 2. 핵심 플로우

### 2-1. 라이브 모드 (스피드 퀴즈)

> **스피드 퀴즈**: 학생이 자기 속도로 전체 문항을 개별 진행. 교사가 문항을 넘기는 동기 모델이 아님.

```
[Host] 퀴즈 제작 -> 게임 설정 -> QR 생성
                                    |
[학생] QR 스캔 -> 닉네임 입력 (Guest) / 로그인 (Member) -> 대기
                                    |
[Host] 게임 시작 버튼 클릭
                                    |
[학생] 자기 속도로 전체 문항 풀기 (개별 진행)
                                    |
[Host] 컨트롤 패널에서 실시간 모니터링 (학생별 진행도, 문항별 분석)
                                    |
[전체] 결과/랭킹 화면
                                    |
[Host] 리포트 분석 (문항별/학생별)
```

### 2-2. 과제 모드

```
[Host] 퀴즈 제작 → 과제 보내기 (기간 설정) → 이메일 알림/링크 공유
                                    ↓
[학생] 기간 내 자율 접속 → 문항 풀기 (이어하기 가능)
                                    ↓
[학생] 최종 제출 (1회, 확인 팝업)
                                    ↓
[Host] 제출 현황 확인 → 마감 처리
                                    ↓
[Host] 리포트 분석
```

### 2-3. Host 대시보드 (Sitemap)

```
홈 대시보드
├── 내 자료
│   ├── 내 퀴즈 → 에디터 (퀴즈 만들기)
│   └── 게임 설정
│         ├── [라이브] QR/링크 게임 열기 → 대기화면 → 게임 시작
│         └── [과제] 기간 설정 + 알림 발송
├── 그룹 관리
│   ├── 그룹 목록 (생성/삭제)
│   └── 그룹 상세 (멤버 초대/제거, 게스트 링크)
└── 지난 게임 & 리포트
      ├── 지난 게임
      ├── [과제] 제출 현황 대시보드
      └── 리포트 (문항별 / 학생별)
```

## 3. 진행 방식 비교

| 항목 | 라이브 -- 스피드 퀴즈 (Live) | 과제 (Assignment) |
|------|------------------------------|-------------------|
| 진행 방식 | **학생 개별 속도** (교사는 모니터링) | 학생이 기간 내 자율 진행 |
| 입장 방법 | QR/링크 (즉시) | QR/링크 (오픈 기간 내) |
| 교사 컨트롤 | 일시정지/강제종료만 (문항 넘기기 없음) | 없음 |
| 이어하기 | 없음 | 있음 (UUID / 이메일+닉네임 기반) |
| 제출 횟수 | 1회 (자동) | 1회 (제출 확인 팝업) |
| 랭킹 공개 | 종료 즉시 | 마감 후 공개 (추후) |
| 정답 해설 | 즉시 | 공개 설정에 따라 |
| WebSocket | 필수 | 불필요 (REST만 사용) |

### S-03. 에디터

**라우트** `/sets/[setId]/edit` | **역할** Host

에디터는 4가지 모드를 가짐: `edit` (기본 편집) / `template` (문항 추가 유형 선택) / `import` (문항 뱅크 불러오기) / `ai` (AI 추천)

```
┌─────────────────────────────────────────────────────────────┐
│  [퀴즈명 (인라인 수정)]                    [미리보기] [저장]  │
├────────────────────┬────────────────────────────────────────┤
│  [불러오기][AI추천]  │  문항 편집 (우)                         │
│  [셔플] {N}개       │                                        │
│                    │                                        │
│  1. 광합성은? ◀────┤──▶ 유형: [객관식 ▼]                    │
│  2. 세포란?        │    지문: [문제를 입력하세요]              │
│  3. ···            │    보기: ①____ ②____ ③____ ④____      │
│                    │    정답: [② ▼]                         │
│  [+ 문항 추가]     │    힌트: ____                           │
│  (유형 선택 화면)   │    해설: ____                           │
└────────────────────┴────────────────────────────────────────┘
```

| 기능 | 설명 |
|------|------|
| 문항 유형 | 객관식 (2/4/5지선다) / O·X / 단어 배열(unscramble) |
| 문항 CRUD | 추가, 수정, 삭제, 복제, 순서 이동 (▲▼) |
| 문항 템플릿 | [+ 문항 추가] 클릭 시 유형 선택 카드 UI 표시 (객관식 2지선다, 단답형, 객관식 4지선다, OX 퀴즈, 객관식 5지선다). 미니 프리뷰 카드로 시각적 선택 |
| 문항 불러오기 | 전체화면 모드. 6가지 필터(학년, 과목, 단원, 난이도, 유형, 키워드 검색) → 체크박스 다중 선택 → 하단 액션바에서 일괄 추가 |
| AI 추천 | 전체화면 모드. 과목/학년/단원 + 난이도별 문항 수(쉬움/보통/어려움) + 문항 유형 체크박스 → AI 추천 요청 → 결과 전체선택/개별선택 → 추가. `POST /api/question-bank/ai-recommend` 호출 |
| 셔플 | Fisher-Yates 랜덤 셔플 |
| 정답 미지정 | 저장 시도 시 "정답을 지정해 주세요" 에러 + 해당 문항 자동 선택 |

### S-04/S-09. 게임 설정

**라우트** `/sets/[setId]/deploy` | **역할** Host

> 게임 유형(`game_mode`)은 현재 `tug_of_war`(줄다리기)로 자동 지정됨. UI 선택 불가 (추후 게임 모드 선택 기능 추가 예정).

```
┌─────────────────────────────────────────────────────┐
│  진행 방식    ● 라이브   ○ 과제                       │
│                                                     │
│  ─── 게임 설정 ───                                   │
│  문항당 시간  [ 20 ]초                               │
│  오답 재도전  [토글 스위치]                            │
│  힌트 사용    [토글 스위치]                            │
│                                                     │
│  ─── 과제 설정 (과제 탭 선택 시만 노출) ───            │
│  오픈 기간    [시작일]  ~  [마감일]                    │
│  정답 공개    ● 제출 즉시  ○ 마감 후  ○ 비공개        │
│                                                     │
│  ─── 참여 대상 ───                                   │
│  ● QR 공개                                           │
│  ○ 기존 그룹 선택  [그룹명 ▼]                        │
│  ○ 신규 그룹 생성  [그룹명 입력]                      │
│                                                     │
│              [게임 열기 →]                             │
└─────────────────────────────────────────────────────┘
```

| 과제 전용 | 설명 |
|-----------|------|
| 오픈 기간 | 날짜 지정 or 무기한 |
| 제출 횟수 | 1회만 허용 |
| 정답 공개 | 제출 즉시 / 마감 후 / 비공개 |
| 그룹 선택 시 | 멤버에게 과제 안내 이메일 자동 발송 |
| 게스트 | 링크 복사 (Host가 수동 발송) |

| 기능 | 설명 |
|------|------|
| 방 코드 | `QUIZ-0000` 형태, 복사 가능 |
| 이메일 초대 | 이메일 입력 → 참여 링크 발송 |
| 검색/필터 | 이름 또는 이메일로 참여자 검색 |
| 퀴즈 정보 패널 | 문항 수, 유형별 비율, 제한 시간, 게임 모드, 힌트 허용 여부 |
| 게임 시작 확인 | 0명 시 경고, 미준비자 있을 시 안내 팝업 (AlertDialog) |

### S-07. 컨트롤 패널 (v3 -- 스피드 퀴즈)

```
|   [일시정지/재개] [강제종료]                            [나가기 X]     |
+-----------------------------------------------------------------------+

|  +--------------------------------+  |  +---------------------------+ |
|  | # | 학생  | 진행도| 점수|뱃지 |  |  | 일차방정식 2x+3=7의 해는? | |
|  | 1 |수학천재| 5/5 v | 300 |우수 |  |  | (1) x=1  (2) x=2  ...    | |
|  | 2 |김민준 | 5/5 v  | 280 |    |  |  +---------------------------+ |
|  | 3 |이서연 | 4/5    | 270 |    |  |  +- 응답 분석 ---------------+ |
|  | 4 |박지훈 | 2/5    | 100 |느림|  |  | 정답: (2) x=2             | |

+-----------------------------------------------------------------------+
```

**좌측 패널 (flex-3) -- 학생 모니터링**

| 컬럼 | 설명 |
|------|------|
| 순위 (#) | 점수 desc -> 총 응답시간 asc 기준 |
| 학생 | 닉네임 + 아바타 |
| 진행도 | `3/5` (완료 시 체크 아이콘), 프로그레스 바 |
| 점수 | 현재 누적 점수 |
| 정답률 | 백분율 |
| 뱃지 | 우수/연속오답/찍기의심/진행느림 |

- 뷰 전환: 테이블뷰 / 카드뷰
- 필터: 전체 / 뱃지별 (연속오답, 찍기의심 등)
- 정렬: 순위(기본), 이름, 점수, 정답률
- 다중 선택 -> ReactionBar로 알림 전송

**우측 패널 (flex-2) -- 문항 + 분석**

| 영역 | 설명 |
|------|------|
| 문항 네비 | `<- Q1 / 5 ->` 화살표로 전체 문항 탐색 |
| 문항 내용 | 질문 텍스트 + 선택지 (정답 강조) |
| 응답 분포 | 각 선택지별 선택 수 + 퍼센트 + 바 차트 |
| 학생 표시 | 각 선택지 아래에 해당 학생 아바타 칩 표시 |
| 통계 | 정답 N명 / 오답 N명 / 미응답 N명 |

**헤더 (HeaderControlBar)**

| 요소 | 설명 |
|------|------|
| 경과 시간 | 카운트업 (00:00부터 증가) -- 카운트다운 아님 |
| 완료 현황 | `18/24명 완료` |
| 일시정지/재개 | 토글 버튼 |
| 강제종료 | AlertDialog 확인 후 종료 |
| 나가기 | router.back() |

**v3에서 제거된 기능** (v2 동기 모델용):
- 건너뛰기, 다음 문항 버튼
- 힌트 공개 버튼
- 시간 연장 (+15s/+30s/+60s)
- Q번호 뱃지 (개별 진행이라 의미 없음)
- 카운트다운 타이머

**학생 뱃지 (BehaviorBadge)**

| 뱃지 | 조건 |
|------|------|
| `star` (우수) | 정답률 높고 속도 빠른 학생 |
| `struggling` (연속 오답) | 연속 오답 2회 이상 |
| `guessing` (찍기 의심) | 빠른 응답 + 낮은 정답률 |
| `slow` (진행 느림) | 전체 문항 중 절반 미만 완료 (다른 학생이 이미 완료한 상태) |

## 4b. 분석 엔진 (Analysis Engine)

> `frontend/lib/analysis.ts` — 리포트에서 사용하는 클라이언트 사이드 분석 함수 모음.
> 모든 함수는 순수 함수로, API에서 받은 raw 데이터를 입력받아 계산된 결과를 반환.

### 함수 목록

| 함수 | 입력 | 출력 | 용도 |
|------|------|------|------|
| `calcQuestionSummary` | questions, events, totalParticipants | `QuestionReportSummary` | 종합 요약 (제출율/평균점수/시간/Top3/경고) |
| `calcQuestionBars` | questions, events | `QuestionBarData[]` | 문항별 정답률 가로바 데이터 |
| `calcConceptUnderstanding` | questions, events | `ConceptUnderstanding[]` | learning_map 기준 개념별 이해도 |
| `calcStreaks` | events (학생 1명) | `{ maxCorrect, maxWrong }` | 연속 정답/오답 수 |
| `calcStudentPattern` | studentEvents, avgTimeSec | `AnalysisPattern` | 학생 1명의 2축 패턴 (이해x성실) |
| `getPatternLabel` | AnalysisPattern | `PatternLabel` | 패턴 → 한글 라벨 변환 |
| `getCoachingLabel` | AnalysisPattern | `CoachingLabel` | 패턴 → 코칭 라벨 변환 |
| `calcAllStudentAnalyses` | results, events, totalQuestions | `StudentAnalysis[]` | 전체 학생 분석 일괄 계산 |
| `calcPatternDistribution` | analyses | `Record<PatternLabel, number>` | 패턴별 학생 수 분포 |
| `calcCoachingTopN` | analyses, label, n | `StudentAnalysis[]` | 특정 코칭 라벨 Top N 추출 |

### 패턴 판정 임계값

```
이해 (understands): 정답률 >= 60%
성실 (diligent):    학생 평균시간 >= 전체 평균시간 * 50%
도움 필요:          연속오답 >= 3 또는 정답률 < 40%
칭찬 필요:          이해 + 성실 + 연속정답 >= 3
```

### 타입 정의 (types/index.ts)

```typescript
type PatternLabel = '이해' | '미이해' | '성실' | '찍기'
type CoachingLabel = '도움 필요' | '칭찬 필요' | '관찰' | '양호'
type ResponseEventType = 'first_input' | 'modify' | 'final_submit' | 'skip' | 'return'

interface AnalysisPattern {
  understands: boolean
  diligent: boolean
  needsHelp: boolean
  needsPraise: boolean
}

interface StudentAnalysis {
  nickname: string
  participantType: ParticipantType
  totalScore: number | null
  correctCount: number | null
  totalQuestions: number
  pattern: AnalysisPattern
  patternLabel: PatternLabel
  coachingLabel: CoachingLabel
  streaks: { maxCorrect: number; maxWrong: number }
  rank: number | null
}
```

```
|  | 10문항 . 객관식6 OX4 . 쉬움                           ||
|  | [9수01-03]                                           ||
|  | by 박수학 선생님 . 1달 전                              ||
|  +------------------------------------------------------+|
|                                                          |
|  [< 1 2 3 ... >]  페이지네이션                            |
+----------------------------------------------------------+
```

**기능:**
- 필터: 과목, 학년 (단원·성취기준·문항유형 필터는 미구현, 추후 추가 예정)
- 정렬: 인기순(좋아요+다운로드), 최신순, 좋아요순, 다운로드순
- 페이지네이션: 12개/페이지
- 카드 클릭 시 상세 페이지 이동

- 신고 다이얼로그 (사유 선택)

### 공유 설정 다이얼로그

> 내 퀴즈(S-02) 또는 에디터(S-03)에서 "커뮤니티에 공유" 선택 시 표시

```
+---------------------------------------------+
|  커뮤니티에 공유하기                    [x]   |
|                                             |
|  제목 *   [________________]                |
|  설명     [________________]                |
|  과목 *   [수학 v]                          |
|  학년 *   [중1 v]                           |
|  태그     [태그 입력 후 Enter] [추가]        |
|           [소수 x] [합성수 x]               |
|                                             |
|  [취소]              [공유하기]              |
+---------------------------------------------+
```

**필드:**
- 제목 (필수): 기본값 = 원본 퀴즈 제목
- 설명 (선택): Textarea 자유 텍스트
- 과목 (필수): 드롭다운 (수학, 영어, 과학, 사회, 국어, 기타)
- 학년 (필수): 드롭다운 (초1~고3)
- 태그 (선택): Enter 키 또는 [추가] 버튼으로 개별 추가, Badge로 표시 + 삭제

> **미구현 필드 (추후 추가 예정):**
> - 단원: 과목/학년 선택 시 연동
> - 성취기준: 검색형 멀티셀렉트 (`[9수01-01]` 형식)
