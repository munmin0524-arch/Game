# 학생 복습 플로우 종합 설계

## Context
학생 복습 화면(`/student/*`)의 사용자 플로우가 불완전함. 오답노트, 추천 학습, 보상 시스템, 게임 이력 등 핵심 기능이 누락. GNB 구조 재설계 필요. SPEC.md에 학생 복습 관련 스펙이 없어 새로 정의.

---

## 1. GNB 재설계: 5탭 → 5탭 (역할 재정의)

### 현재 문제
- "게임하기"와 "홈"의 역할 중복
- "학습보상" 독립 탭이기엔 얕음 (목표 카드 5개 뿐)
- "오답노트"가 학습보상 하위에 숨어 접근성 부족
- "캐릭터"와 "보상"이 별도 탭일 필요 없음

### 확정 구조: 홈 / 학습 / 보상·꾸미기 / 학습이력 / 분석결과

| 탭 | 라벨 | 아이콘 | 경로 | 포함 기능 |
|---|---|---|---|---|
| 1 | **홈** | Home | `/student` | 오늘의 할일 3대 미션, 진도 현황, 곰젤리 잔액, 간단 요약 통계 |
| 2 | **학습** | Gamepad2 | `/student/learn` | 게임 모드 선택(NPC대전/혼자풀기), 오답노트 복습, 자동매칭/직접선택 |
| 3 | **보상·꾸미기** | Star | `/student/rewards` | 곰젤리 잔액 + "더 받으러가기", 보상 획득 리스트, 캐릭터 초대·꾸미기 |
| 4 | **학습이력** | ClipboardList | `/student/history` | 날짜별 게임 목록, 세션 유형별 필터(일반/추천/오답), 학습맵(단원별 숙련도), 뱃지 |
| 5 | **분석결과** | BarChart3 | `/student/my-result` | 요약 통계, 학습맵 탭, 주간/월간 트렌드, 뱃지, 캐릭터 |

### 홈에 대시보드 요약 포함
- 별도 대시보드 없이 홈 하단에 간단 요약: 풀은 문제/정답률/출석연속/달성목표
- 상세 통계는 "분석결과" 탭에서
- 게임 이력은 "학습이력" 탭에서

### 각 탭 콘텐츠 밸런스
- **홈**: 미션 3개 + 진도 4칸 + 젤리 잔액 = 스크롤 1~1.5회
- **학습**: 게임모드 선택 + 오답노트 CTA + 게임카드 캐러셀 = 스크롤 1~1.5회
- **보상·꾸미기**: 젤리 잔액 + 획득 리스트(5~8개) + 캐릭터 초대 + 꾸미기 = 스크롤 1.5회
- **학습이력**: 게임 이력 목록 + 학습맵 + 뱃지 = 스크롤 2회+
- **분석결과**: 종합 통계 + 트렌드 차트 + 학습맵 = 스크롤 2회+

**수정**: `components/student/StudentNav.tsx`

## 3. 진단 및 추천 알고리즘

### 난이도 기반 출제 삭제
현재 문항이 모두 "쉬움" 수준이라 난이도(쉬움/보통/어려움) 기반 적응 출제는 의미 없음.
대신 **단원별 정답률 + 풀이 횟수** 기반으로 추천.

### 단원별 숙련도 4단계 (내부용, 학생에게 레벨명 노출하지 않음)

| 레벨 (내부) | 조건 | 추천 액션 | 학생 표시 |
|---|---|---|---|
| mastered | 정답률 ≥ 85% AND 10문제+ | 유지 (복습 권장만) | 🟢 정답률% + "잘하고 있어요!" |
| proficient | 정답률 ≥ 65% AND 5문제+ | 7일+ 미학습 시 추천 | 🔵 정답률% + "좋아요, 계속!" |
| developing | 정답률 ≥ 40% OR 1~4문제 | 적극 추천 (취약) | 🟡 정답률% + "조금만 더!" |
| beginner | 풀이 0 OR 정답률 < 40% | 최우선 추천 | ⚪/🔴 "--" 또는 정답률% + "아직 안 배웠어요"/"도전해봐요!" |

학습맵에서 과목 드롭다운(또는 탭)으로 전환, 대단원 > 중단원 트리 구조로 표시.
각 중단원마다 정답률 바 + 색상 + 격려 메시지 + [학습하기 →] 버튼(약한 단원만).

### 추천 유형 3가지
1. **미학습 단원**: `totalAttempts === 0`, 교육과정 순서대로 우선순위
2. **취약 단원**: developing/beginner이면서 풀이 1+, 정답률 낮은 순
3. **복습 권장**: proficient이지만 7일+ 미학습

### 출제 로직
- 선택된 단원(대단원/중단원)에서 10문제 랜덤 추출
- 이전에 틀린 문제 우선 포함 (최대 3문제)
- 나머지는 안 풀어본 문제 우선

### 복습 vs 추천의 차이
- **복습(오답노트)**: 문제 단위. 이전에 틀린 바로 그 문제를 망각곡선에 따라 반복
- **추천 학습**: 단원 단위. 약한 단원의 새 문제(pool에서 10문제)를 풀어 실력 올리기
- 같은 추천 로직, 다른 진입점: 홈에서는 시스템 자동 픽, 학습맵에서는 직접 선택

**신규**: `lib/recommendation.ts`
**신규 DB**: `student_unit_mastery` (집계 캐시 테이블)

## 6. 게임 결과 화면 (Blooket 참고)

현재: 결과 → "다음" → `/student/rewards`만 존재
목표: 게임 직후 결과 확인 + 복습 연결 + 학습이력에서 다시 접근 가능

### 게임 결과 화면 = 학습이력 상세 (동일 화면 재활용)

게임 끝난 직후에도 보이고, 나중에 학습이력에서 클릭해도 같은 화면이 뜸.

```

│ ┌─ Q2. 훅의 법칙에서 k는? ────────────┐  │

│ [📖 오답노트에 자동 추가됨 ✅]             │
```

### 핵심 포인트
- **정답률 도넛 차트**: 초록(정답)/빨강(오답) 비율 시각화
- **4칸 요약**: 정답수, 오답수, 총문제, 획득 젤리
- **문항별 상세**: 모든 선택지 표시 + 내가 뭘 골랐는지 + 정답 표시 + 소요시간
- **오답에만 해설 펼치기**: 정답 문항은 접혀있고, 오답 문항은 해설 버튼
- **오답 자동 추가**: 게임 끝나면 틀린 문제가 자동으로 오답노트에 들어감
- **학습이력에서 재접근**: 같은 URL(`/student/game-result/[sessionId]`)로 나중에도 조회 가능
- 학습이력에서 접근 시에는 하단 액션 버튼 대신 [오답노트에서 복습하기] 링크만 표시

### 세션 유형 태그
결과 화면 상단에 어떤 종류의 학습이었는지 표시:
- `🎮 일반 게임` — 직접 선택 또는 자동매칭
- `💡 추천 학습` — 추천 알고리즘으로 시작한 게임
- `📖 오답 복습` — 오답노트 복습 세션

**수정**: `app/student/game-result/[sessionId]/page.tsx` (전면 재작성)
**신규**: `components/student/ResultDonutChart.tsx`, `components/student/QuestionResultCard.tsx`

### 핵심 포인트

## 9. 게임 모드: NPC 대전 vs 혼자풀기

기존 `game-play`에 `mode` query param 추가:
- `?mode=npc` (기본) — NPC 대전, 타이머 20초, 점수 비교
- `?mode=solo` — 혼자풀기, 타이머 없음, 해설 즉시 표시
- `?mode=wrong_note` — 오답노트 복습 (혼자풀기 + SR 스케줄 연동)

혼자풀기 모드는 NPC/점수바 없이 문제 → 정답 확인 → 해설 → 다음 문항 흐름.

**수정**: `app/student/game-play/[sessionId]/page.tsx` — mode 분기
**신규**: `components/student/SoloPlayView.tsx`
